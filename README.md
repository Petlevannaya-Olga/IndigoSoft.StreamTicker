#Тестовое задание "Система агрегации и обработки данных с биржевых торгов"

Описание:
Разработать имитацию системы сбора, обработки и хранения ценовых данных с нескольких бирж в реальном времени. Нагрузка: до 50-100 тиков/сек суммарно.

Функционал:
### 1. **Сбор данных**:
-Клиент WebSocket (real-time stream)  2-3 клиента одновременно подключаются к серверным вебсокетам и получают поток котировок (тикер, цена, объем, timestamp) - формат может быть разным

### 2. **Обработка потока данных**:
- Нормализация к единому формату (тикер, цена, объем, timestamp)
- Удаление дубликатов

### 3. **Хранение в БД**:
- Raw данные (тики)
### 4. **Базовый мониторинг**:
- Логирование основных событий (подключение/отключение источника, ошибки)
- Простой счетчик обработанных тиков (в консоль или лог)



Требования:
- Обработка 50-100 тиков/сек (2-3 источника)
- Параллельная работа с несколькими WebSocket подключениями
- Чистая архитектура, SOLID принципы
- Обработка обрывов соединения и автоматическое переподключение

Что оценивается:
- Многопоточность и асинхронная обработка
- Работа с WebSocket соединениями
- Производительность и эффективность работы с БД
- Расширяемость архитектуры для добавления новых бирж
- Обработка ошибок и отказоустойчивость
- Покрытие тестами базовых сценариев
